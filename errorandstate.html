Illustration of some special conditions or errors present in UART

  • Overrun
  • Underrun

Overrun

Overrun error happens when another incoming packet (a byte of data) reaches even before the receiving UART process the previous packet, or in other words, not yet read finished from the receiving UART's buffer. Different devices have various amounts of buffer space to hold the data information. The CPU controller must service the UART in order for the bits in the buffer space to be removed. When the CPU controller missed to service the UART fast enough, resulting in a full buffer, an overrun error will occur, and further incoming information will be lost.

Underrun

Underrun error happens when data transmitted failed to reach at the required time. Output data is to be stored in a buffer usually, then it will be fetched and processed at a predetermined time. However, if the transmit buffer is empty when the fetch function is called, and the next packet cannot be output, underrun error will occur.

Break

A "break" condition happens when the transmitting UART holds the transmitting data line at low logic state for a long duration, i.e. more than the time needed to transmit the whole packet, when the idle state is at logic one. This could happen as well when the speed of transmission from the transmitter is lower than what the receiver is expecting. However, such mismatched may not necessary be an error, but as a useful way to signal the receiver to rectify the situation.

Parity Error

Parity error happens when the parity of the total number of one-bits is incorrect when compared to what the parity bit specifies. By checking parity, transmission errors can be detected as often. This could possibly caused by data corruption, bits being flipped during transmission, wrong baud rates between UARTs, etc. Note that it is optional to use a parity bit, thus parity error will only happen when it is enabled.

Framing

Framing error happens when the UART does not read a stop bit at the end of a packet or at its expected time. As the start bit serves as an identifier of the start of a data packet, its timing is a reference for the rest of the bits. If the transmission data line is not at its expected logic state when it anticipates the stop bit (due to the number of bits in the data frame, as well as the number of parity bits for which its set for the UART), the UART will produce a framing error.
